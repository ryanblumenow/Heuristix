Data science aspirants doubts regarding A/B testing

Step-by-step, beginner-friendly tutorial on A/B testing for Data Science aspirants.

In this article, we are going to discuss what is A/B testing?, Where you can implement and more key concepts of A/B testing in data science.

In the last decade, data science has become a popular and effective technique to solve business problems. One of the business problems is “Will the user like the new version or not?” For example, if you build the app for users and later you want to integrate a new feature into the app but you are not sure about this so you want to make a decision whether to roll out the feature or not. Here you can take the user’s input whether they want a new feature or not so that you will gain more traffic.

Now this problem can be solved by conducting an experiment/test and that test is known as A/B testing. Let’s start.

What is A/B testing?

A/B testing is a randomized experiment where it aims to compare two variants/versions (A vs B) of something (webpage, images, etc.) to determine which variation performs better for a given conversion goal based on user experience.

It is also known as split testing where you randomly split the users into a control group and treatment/variation group in which one version is shown/given to one group and another version to another group then based on user input, you determine which version is better or performing well.

In statistics, an A/B test is an example of statistical hypothesis testing, a process whereby a hypothesis is made about the relationship between two data sets and those data sets are then compared against each other to determine if there is a statistically significant relationship or not.

Variants of A/B testing:

A/B testing is comparing two versions, but what if you want to compare more than two versions.

But as a data scientist, sometimes we have to test more than two versions. If you are testing three versions then the testing will be named A/B/C testing. In general, for n versions, it will be A/B/n testing. The procedure will be the same as A/B testing.

Where A/B testing is used?

For A/B testing, the experimenter needs the user’s input to do the experiment and the web page is the online way to do it. You might be thinking, you can also go to an individual user to get the input, yes you can do but through this, your sample size will be too small. Hence your sample can’t be equal to the true population.

It helps in making decisions because it estimates population parameters based on sample statistics.

Why data scientists do A/B testing?

A/B testing helps organizations to avoid unnecessary risks by allowing them to target their resources for maximum effect and efficiency. Also, it helps them to select the best version of products (best webpage, best images, best marketing promotional content, etc.)

How long should you run an A/B test?

To get representative sample data and for your data to be accurate, experts recommend running the A/B test for a minimum of one to two weeks.

When not to run A/B testing?

  1. When the sample size is small: When you don’t have enough meaningful traffic.
  2. If you don’t yet have an informed hypothesis: Identify your problem. Define a hypothesis. Then test to see if you’re right. For example, the Problem Statement: Current promotional content is not doing well. Hypothesis:- If I add some more visuals then more people will look into it.
  3. If you can’t spend time: Suppose you can do the experiment but if you can’t spend time on analyzing the data or testing the hypothesis, doing A/B is useless.

When to run A/B testing?

  1. When the sample size is large.
  2. You can do A/B testing when you want the user’s input to fulfill your goal.

Advantages of A/B testing?

  1. Help in decision making: If you have two versions of the same product then A/B testing will help you to select the best one.
  2. Reduces risk: You can avoid/reduce the risk. Let me explain to you through an example: Suppose you made some major changes on a web page and before rolling out for all the users, you can check whether the new changes will be liked by the users or not.

How to Conduct A/B Testing?

If you are a data scientist and wants to conduct A/B testing for your organization then follow the below steps:-

Step 1) Collect data:

Before comparing two versions, you must data for the 1st version or both.

Suppose you want to compare two versions of a webpage then you must have data for the first webpage then through testing you will collect data for the 2nd version.

Suppose you want to compare two promotional campaigns, then obviously you don’t have the data for the 1st version also. In that case, you will collect data for both the version through A/B testing.

Step 2) Identify goal:

Identify the problem. Define the hypothesis then test to see if you’re right. Defining the goal will help in understanding what the objective is.

Step 3) Generate hypothesis:

Once the goal is set for the A/B test. The next step is generating the hypothesis i.e how to improve the original version and how to make it better than the current version.

For example, You work at an e-commerce company, you think that displaying the checkout button on the product page can increase the checkout rate.

Step 4) Determine sample size:

To do the A/B test, you need to know in advance how many users have to visit the page so that the test results are statistically significant.

Based on this, it will be decided how long you have to run the A/B test to collect the data from users.

Step 5) Run the experiment:

The next step is to run the A/B testing. Once the sample size is defined you have to split your versions equally and randomly.

Example: You have defined a sample size of 20,000 users and have to compare two versions of the web page. Next, you have to split equally & randomly your 20k users into two groups such that each version will be shown to 10k users.

Step 6) Analyze the data:

Once this experiment is complete, as a data scientist the next step is to analyze the results. The data collected from the experiment and will tell you the difference between the performance and efficiency of different versions of the same product. It will also show whether there is a significant difference between the variables using statistical data. This will help you to make decisions such that you have to go with version 1 or 2.

Practical example:

Suppose you are a data scientist at Towards Data Science and a company released the Bootcamp for the data science aspirants and later on you identified key features that to be added in Bootcamp and you have come up with two versions as follow:

Hypothesis: If we give the guaranteed job and paid internships then more data science aspirants will join the Bootcamp.

Determine sample size: Suppose weekly visitors at Towards Data Science are nearly 50k and you have decided to split both the versions equally to 20k users.

Run the experiment: You run the experiment for two weeks and collected the data like total visitors, out of total visitors who saw the content than out of them who enrolled into Bootcamp.

Analyze the data: Once the experiment is completed, you start analyzing the data based on user experience.

You have received the above data after running the A/B testing.

Total 8k users went through the Bootcamp page of version A and out of them only 1000 students enrolled in the Bootcamp.

Total 5k users went through the Bootcamp page of version B and out of them, 2000 students enrolled in the Bootcamp

Since fewer users saw version B but 40% of users are enrolled in the Bootcamp.

Hence, after A/B testing Towards Data Science can make the decision whether to addon new features or not.

Thanks for reading. I hope this helped you to understand the core concepts of A/B testing in data science.
